Cybersecurity Awareness Hub – Project Overview

Project Overview

This web app aims to serve as an educational hub to help students and everyday internet users learn the basics of cybersecurity.

  • Intended Users: College students, teachers, general internet users interested in cybersecurity
  • Content Overview: Educational pages about cybersecurity, interactive quizzes, safety tips, downloadable resources, and a form to contact or ask questions

Page Design

  • Home (index.html):
    • Purpose: Introduce the site, share a welcome message, and link to main sections
    • Audience: All users
    • Content: Banner, intro text, links to other sections
    • Interactivity: Navigation bar, interactive welcome animation
  • About Cybersecurity (about.html):
    • Purpose: Explain cybersecurity fundamentals
    • Audience: Beginners
    • Content: Definitions, why it's important, types of threats
    • Interactivity: Accordions or toggles to expand definitions
  • Tips & Best Practices (tips.html):
    • Purpose: Share practical advice for staying safe online
    • Audience: All users
    • Content: Password tips, recognizing scams, software to use
    • Interactivity: JavaScript to toggle "More Info", security quiz
  • Resources & Tools (resources.html):
    • Purpose: Provide useful links and tools
    • Audience: Students and professionals
    • Content: Downloadable PDFs, links to security tools
    • Interactivity: Gallery, buttons with hover effects
  • Contact / Feedback (contact.html):
    • Purpose: Allow users to submit questions or feedback
    • Audience: All visitors
    • Content: Contact form (name, email, message)
    • Validation: All fields required
    • Interactivity: JavaScript form validation

Dynamic Functionality

Planned Interactive Features:

  • Interactive Quiz (Tips & Best Practices Page): Engage users with multiple-choice questions about online safety practices. Example: StaySafeOnline’s safety quizzes
  • Form Validation (Contact Page): JavaScript-powered form validation to ensure proper input for name, email, and message fields. Example: Standard web form validation behavior
  • Phishing Gallery Viewer (Resources Page): Visual examples of phishing attempts presented in a lightbox-style image gallery. Example: jQuery Lightbox or similar viewer plugin
  • Collapsible Content (About Page): Toggle definitions and terms with an accordion UI for beginner-friendly learning. Example: jQuery UI Accordion
  • Dynamic Navigation Elements (All Pages): JavaScript-enhanced elements for smooth navigation, animations, and user engagement. Example: Interactive landing page animations
